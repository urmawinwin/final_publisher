float이란?
  - float은 객체를 좌측으로 정렬하는 기능으로, 레이아웃 아웃라이너 잡을때 많이 쓰이입니다.
  - float기능을 쓴다면, 반드시 부모는 float 해제를 시켜야 합니다
  - float해제 이유는, float 기능을 쓰면 height 값이 없어지기에 레이아웃이 깨지기 때문입니다.
  - 해지 방법중 간단한것은, 부모태그에 overflow:hidden 을 주는것이나
  - float을 해제하는 방법은 많으나 제일 권장되는 방식은
  - float된 요소의 부모태그에 가상요소를 만들고 해당요소에 clar:both를 지정해줍니다.
  * claer 속성
  - clear:none // 기본값으로 clear를 설정하지 않은 것과 같습니다.
  - clear:left // 왼쪽으로 붙는 float 정렬을 취소합니다.
  - clear:right // 오른쪽으로 붙는 float 정렬을 취소합니다.
  - clear:both // 오른쪽,왼쪽 모두 붙는 float 정렬을 취소합니다.

  /*float해제 : 부모태그에 overflow:hidden 이 쓰인 경우*/
  <.parent {
    overflow: hidden;
    padding: 30px 0;
  }>
  <.parent > div {
    float: left;
    width: 40%;
  }>

  /*float해제 : 부모태그에 가상요소 ::after 이 쓰인 경우*/
  <.parent {
    content:'';
    display: block;
    clear:both;
  }>
  <.parent > div {
    float: left;
    width: 40%;
  }>


css 순서
  -웹사이트 작업시 하나의 태그에 여러 가지의 CSS가 적용될 때가 있습니다.
  -이 때 특정한 규칙에 따라 CSS에 우선 순위가 부여되어 적용됩니다.
  -이 규칙을 CSS 우선 순위라고 합니다.

  -기본적으로 뒤에 나오는 css가 우선순위가 높습니다. 
  -!important > inline style attribute > id > class, 
  -다른 attribute, 수도클래스(:first-child같은 것) > tag element,
  -수도엘레먼트(::before같은 것) 순으로 우선순위가 높습니다.
  -우선순위가 같다면 개수가 많은 css가 우선순위가 높습니다.

반응형사이트

미디어쿼리를 사용하여 작업을 완료한 반응형 사이트입니다.
다양한 디바이스의 요구가 높아지는 트렌드를 반영하여 데스크탑,아이패드,
핸드폰 여러 기종의 사이즈를 준수하였습니다.

미디어쿼리란?
  미디어 쿼리(mediaqueri)는 사이트에 접속하는 장치에 따라
  특정한 CSS 스타일을 사용하도록 도와주는 소프트웨어 모듈입니다.
  미디어 쿼리를 이용한 사이트는 웹 사이트에 접속하는 기기에 따라서
  레이아웃이 바뀌게 됩니다. 즉, PC로 접속하면 모니터 화면에 맞게
  스마트폰으로 접속하면 스마트폰 화면에 맞게 레이아웃이 변경되는 것이지요.
  웹 문서의 스타일을 정의할 때, 즉 CSS 코드를 작성할 때
  미디어쿼리 모듈을 사용해 주는 것이 일반적인 사용 방법입니다.

  /*style*/
  @media screen and (min-width: 200px) and (max-width: 360px) { /* 스타일 정의 */}
Sprite란?
  -대부분의 웹 서비스에서는 레이아웃 구성에 필요한 다수의 이미지(버튼이미지,아이콘 등)를
  -하나의 이미지에 포함 시키고, 좌표와 크기 지정을 통해 추출하여 사용하는 방식을 선택하고 있습니다.
  -이런 목적으로 만들어진 이미지 리소스를 스프라이트(Sprite)라고 합니다.
  -낱장의 이미지보다 1장의 이미지 파일을 웹문서를 전송하는 것이기에
  -즉, HTTP 요청(Request) 횟수를 줄일 수가 있습니다.
  -위치는 X와 Y의 픽셀좌표를, W는 폭(With)을, H는 (Height)를 의미 합니다.

  /*html*/
  <div id="container">
    <div class="#sprite"></div>
  </div>

  /*style*/
  #container{
    position:relative;
  }
  .sprite{
    position:absolute;
    display:block;
    width:52px;
    height:52px;
    background:url("spr_image_01.png");
    background-position:-136px -66px;
    margin:0 auto;
  }
          
Retina란?
  - 디바이스는 물리적인 기계로 스마트폰, 데스크탑, 랩탑 등과 같은 것들이 디바이스에 해당됩니다.
  - 디바이스마다 픽셀 밀도가 달라지게 되었는데 이 시작이 2010년의 애플사가 발표한
  - 디스플레이에서 비롯됩니다. 애플(Apple)사가 2010년 inch 당 픽셀을 x2 배로 올려서
  - 엄청나게 선명한 레티나 디스플레이를 소개한 이후,
  - 그래픽 디자인과 설계 과정에서 디바이스의 픽셀을 고려해야 하는 상황에 직면하게 됩니다.
  - 이렇게 레티나 디스플레이 발표 이후에 x3(3배), x4(4배)로 계속해서 늘어나게 되었으며,
  - 이에 따라서 비주얼(CSS 포함) 디자인 과정도 복잡해지게 되었습니다.
  - 디바이스 픽셀 밀도에 영향을 받는 것 중에는 jpg, png, gif 비트맵 데이터의 경우에 주의하여 그래픽 제작이 필요하고,
  - 벡터 그래픽의 SVG 나 CSS 같은 것은 디바이스 픽셀 밀도에 영향을 받지 않는다 는 점을 기억해야 합니다.

  -SVG 나 CSS 와 같은 벡터 그래픽은 디바이스 픽셀 밀도에 영향을 받지 않기 때문에 CSS 제작자는 비율 1만 고려하면 되지만 jpg, png, gif 와 같은 비트맵 데이터를 CSS 로 제어할 경우에는 배수 x2, x3, x4 등을 고려해서 작성해야 합니다.
  예를 들어 사진 같은 경우에 jpg 로 제작이 되는데 이를 CSS 로 제어할 때 고려해야 한다는 것입니다.
  물론 사진 아닌 그 외의 제작된 그래픽이 SVG 로 제작된다면 CSS 제어 필요성도 없어지게 될 것입니다.
한줄효과란?
  - 긴 문장을 줄여 한줄로 보이게 하는 효과 입니다.
  - 게시판 게시글 내용 또는 제목에서 자주 사용되어집니다.

  /*html*/
  <ul class="menu">
    <li class="name_one">전체 메뉴 바로가기</li>
    <li class="name_one">배너 영역 바로가기</li>
    <li class="name_one">컨텐츠 영역 바로가기</li>
  </div>

  /*style*/
  .name_one{
    width:160px;
    with-space:nowrap;
    overflow:hidden;
    text-overflow:ellipsis;
  }

크로스 브라우징이란?
  - 웹 페이지 제작 시에 모든 브라우저에서 깨지지 않고 의도한 대로 올바르게(호환성) 나오게 하는 작업을 말한다.
  - [HTML, CSS, Javascript 작성 시 W3C의 웹 규격에 맞는 코딩을 함으로써
  - 어느 브라우저, 기기에서 사이트가 의도된 대로 보여지고 작동되는 기법.]
  - 브라우저 마다 랜더링 엔진이 다르기 때문입니다.

  *크로스 브라우징 작업
  - 1. 도움이 되는 사이트를 이용하여 브라우저에 맞게 작업한다. - 캔아이유즈 https://caniuse.com

  - 2. 초기화 작업 (CSS 초기화 작업) - 브라우저마다 차이가 나는 기본 스타일 값들을 초기화 시킨다.

  - 3. 핵(Hack)
  - 스타일을 줄 때 특수문자를 넣어서 다른브라우저들에서는 인식이 안되지만
  - IE 특정 버전에서는 인식되도록 하는 방법이다. 이 방법은 추천하지 않는다.
  - IE8 이상의 브라우저에서 구분해내기가 어려우며 CSS 유지 보수를 어렵게 한다.
    div{ color:red; /* IE7이하 대응 */ *color:blue /* IE6 대응 */ _color:green }

  - 4. IE용 주석을 이용한 방법 (Conditional comments)
    HTML 문서 내에서 주석을 이용하여 구별한다.
    [if IE 7]  ![endif]
    [if IE 8] 
 이 문구는 IE8이 포함되지 않은 하위 브라우저, 즉 IE7,6에서 보여지게 됩니다.
 ![endif]-->

    5. 메타 태그를 이용한 IE모드
     // ...  //... 
     요소 안에 넣어두며 IE가 문서를 읽고 랜더링 할 때 원하는 모드로 랜더링을 하게 해준다.
    만약 content에 값이 "IE=edge"라면 해당브라우저가 할 수 있는 가장 최신의 모드로 랜더링한다.
유튜브API란?

협동심

저는 일을할때 제일 중요시 하는것중 하나가 협동심 입니다. 협동심의 동의어는 피드백 이라고 생각합니다. 왜냐하면 주어진 일의 진행사항과 이슈를 올바르게 소통을 해야 나중에 큰 문제가 생기지 않는다고 생각하기 때문입니다. 다양한 경험을 겪으면서 느낀것은, 일을 할때 소통이 잘 이뤄져야 오해 또는 문제가 생기지 않는것 같습니다. 그렇기에 일을 하기전 가장 먼저 갖춰져야할 태도는 팀원들간의 소통x협동심x피드백이 바쁜 일정으로 소통을 주고 받기가 힘들더라도 제일 첫번째로 가져야 할 자세라고 생각합니다.

책임감

책임감을 가지고 결과물을 내는것과, 책임감이 없는채 결과물을 내는것은 퀄리티가 다르다고 생각합니다. 일을함에 있어 이러한 책임의식은 사소할수도 있지만 결국엔 큰 차이를 만든다고 생각합니다. 이러한 책임의식이 바쁜 일정 속에서 문제들을 만나더라도 해결책을 만들어내어 결론적으로 마무리를 잘 짓게 만들어주는 수단이라고 생각합니다.

성실함

반복적일수도 있는 업무를 나태하지 않게 만들어주는것이 바로 성실한 자세라고 생각합니다. 업무가 손에 익다보면 머리가 아닌 몸이 자동으로 일을 하는 경우가 있습니다. 이 경우 쉽게 나태해지며 새로운 지식을 모색하지 않고, 그동안 축적된 지식만을 사용하고 있는 제 모습에 깜짝 놀란적이 있습니다. 그 후 비슷해 보이는 일일지라도 조금이라도 다른점과 새로운 방법을 가지려는 성실한 태도가 정말 중요하다는것을 하게 되었습니다.

열정감

마지막으로 일을 하는데 있어 중요한 4번째 태도는 열정감 이라고 생각합니다. 돌보지 않는 화분에 꽃이 피지 않듯이, 열정이 없이 일을 하는것은 저 본인의 자기발전과 더불어 회사의 발전 모두에 마이너스 효과인것 같습니다. 특히 트렌드 회전율이 빠른 it 분야에서 열정이 없다면 벅찰수도 있기에, 일에 열정을 가지고 잘해보려는 태도를 유지하는게 필수인것 같습니다.
